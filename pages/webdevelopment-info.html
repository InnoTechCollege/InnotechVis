Welcome to Web Development!

Web Development Terms +

Before we get into web development, let's go over some basic termanology used in the web development industry:

  • Server: A computer that is responsible for running all of your back-end needs. These needs can include a database, web server software, your application software and more
  • Client: A computer that makes requests to your server asking for content. This is the role you play when browsing the internet
  • Database: Software that is responsible for efficiently storing and providing data like usernames, password, and other information permanently on a server
  • Web Server: Software that is responsible for allowing your server to listen for requests from clients
  • Internet/Network: An general term for all the connections that facilitate a client request getting to your server

Full-Stack Development +

Web development is a very large and expansive topic. In general it can be split into two different sections, front-end and back-end. Both of these sections are broken down further into more sections that make up the whole of web development.

When put together back-end and front-end programming make up the full-stack. When you claim to be a full-stack web developer, you should have no problem delivering an entire web application to a client.

This means you can configure the back-end from sratch, write the back-end application code and write the front-end application code.

A technology stack is generally made up of the following:

  • The operating system running on your back-end server
  • The software used to run your web server (like Apache or Nginx)
  • The database management software (like MySQL or MongoDB)
  • The development languages and frameworks used to create your website (like JavaScript, VueJS, Python and Django)

A full-stack web developer must have in-depth knowledge of each section in the tech stack being used for an application.

Being a full-stack developer involves knowing how to install software, configure software, troubleshoot issues, code in multiple languages and more.

Front-End Development +

Front-end development is writing code that is designed to run on the client's machine (usually in a browser)

This includes languages like HTML, CSS, JavaScript and the many varients of JavaScript. It also includes Frameworks like VueJS, React, Angular and many more.

Many people assume that front-end code is only for design purposes or for just what the user sees. This is not true. Front-end code today does many complicated things like communicating with API's to update applications in real time.

Back-End Development +

Back-end development is writing code that is designed to run on your server

This includes languages like Pyhton, PHP, Perl, Java, Ruby, and many more. It also includes Frameworks like Django, Laravel, Ruby on Rails and more.

Back-end code is responsible for reacting to responses coming in from the front-end. The request might need to create some new data, get some data, update some existing data, or delete some data. To do this, the back-end will interact with the database and format the data for the user.

Applications that interact with data in this fashion are called CRUD (Create, Read, Update, Delete) applications. A simple examples of this include a web application that allows for users to sign up, log in, update their account or delete their account.
